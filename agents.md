# 🤖 AI 協作歷程 (Agents Log)

這份文件記錄了開發者與 AI 助手在打造「沖繩輕旅」網站過程中的協作歷程與決策思考。

## 階段一：需求分析與資料萃取
- **需求釐清**：使用者提出想要將原本陽春的 Word 檔案，轉換為「好看」、「可透過瀏覽器編輯」且「能與家人分享」的網站。
- **自動化萃取**：AI 助手寫了一段 Python/PowerShell 腳本，直接讀取使用者的 `☀8月沖繩輕旅☀.docx` 內容，並分析出日期、航班、景點等資訊，將其結構化為 `data.json`。這省去了人工手動 key-in 的時間。

## 階段二：設計系統建構
- **美學提案**：針對使用者「現代化美感、配色簡單、創新」的要求，AI 提出了 **Ryukyu Ocean (琉球海洋)** 的設計系統，並實作了毛玻璃 (Glassmorphism) 與動態漸層背景。
- **模組化開發**：AI 自動將設計稿轉換為 1100 行的 Vanilla CSS，並採用 Mobile-first 的響應式設計。

## 階段三：架構重構與決策
- **初始架構 (GitHub API)**：最初 AI 規劃使用 GitHub API 來讓使用者同步資料，並製作了輸入 Token 的 UI 介面。
- **動態調整 (Zero-Friction 協作)**：使用者提出「這樣太麻煩了，讓有連結的人可以修改就好」的要求。
- **解決方案**：AI 迅速轉換思路，在沒有專屬後端伺服器的限制下，捨棄了 GitHub API，轉而串接無需認證的公開 JSONBlob API。這完美解決了安全顧慮與操作門檻，實現了真正的「開箱即用、一鍵同步」。

## 階段四：UI/UX 拋光
- 根據使用者反饋修復了右上角的版型破圖問題。
- 順應使用者的需求，移除了手動同步按鈕，實作了更直覺的「Auto-save」背景自動同步功能。
- 追加了導覽列的「滑動隱藏」效果，提升在手機端瀏覽的沉浸感。
