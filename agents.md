# 🤖 AI 協作日誌 (Agents Log)

此檔案用於記錄與 AI 協作開發 Okinawa 專案時的特殊規則與除錯日誌。

## 協作規則
- **禁止破壞原生事件**：任何 DOM 結構變更必須透過 Vue 的資料驅動 (Data-Driven) 完成，嚴禁使用 `innerHTML` 覆寫以免造成綁定事件遺失。
- **樣式堅持**：維持目前的 Vanilla CSS 變數架構與設計語彙，除非必要，請勿擅自引入 Tailwind CSS 或其他龐大的 CSS 框架。
- **精準排版**：任何 Grid 或 Flex 排版修改，必須考慮手機端與電腦端的 `gap` 計算，避免橫向滾動軸溢出（跑版）。
- **無亂碼寫入**：寫入檔案時需強制使用 UTF-8 編碼，避免繁體中文變成亂碼。
- **部署目錄規範 (GitHub Pages)**：專案打包輸出目錄已配置為 `docs/`（於 `vite.config.js` 的 `build.outDir` 欄位），GitHub Pages 必須設定為透過 `main` 分支的 `/docs` 資料夾進行託管，切勿改回根目錄 `/` 部署，否則會導致原始碼暴露、JS 模組加載 404 及快取衝突。
- **計時器反應性**：倒數計時模組 (`useCountdown.js`) 必須直接 watch `startDate` 與 `endDate` 等基本型別屬性，並開啟 `{ immediate: true }`，以保證非同步資料同步載入後能即時更新，避免首頁計時器卡在「結束了 0 天」的異常狀態。

## 里程碑
- **v1.0**：Vanilla JS + GAS 雲端同步完成。
- **v2.0**：UI/UX 大改版，導入相機膠捲與流暢動畫。
- **v3.0 (2026-06)**：全面遷移至 Vue 3 + Vite 架構，解決長期存在的按鈕點擊失效 bug，並優化響應式排版。
- **v3.1 (2026-06)**：修復倒數計時器載入反應性 Bug，並將 Vite 打包路徑遷移至 `/docs` 成功解決 GitHub Pages 部署白屏問題。
